## Scope

This skill is original documentation and code based on official specifications, package
documentation, package metadata, and model-provider documentation. It contains no bundled
PDF samples, fonts, model weights, language data, or third-party source files. Runtime
fixtures are generated by `scripts/smoke_test.py`.

Package and model facts below were reviewed on 2026-07-16. The pinned requirement files are
the authoritative install inputs. Re-review licenses and provider terms before upgrading a
package or provisioning a model.

## Pinned runtime review

Core direct packages:

| Package | Version | Role | Package license |
| --- | ---: | --- | --- |
| pypdf with `crypto` extra | 6.14.2 | Structure, forms, metadata, pages, AES | BSD-3-Clause |
| pdfplumber | 0.11.10 | Spatial text, words, images, tables, rendering interface | MIT |
| ReportLab | 5.0.0 | PDF creation, overlays, basic AcroForms | BSD |
| Pillow | 12.3.0 | Image validation and fixture generation | HPND/MIT-CMU |

`requirements.txt` pins all resolved transitives with `==`, including cryptography,
pdfminer.six, pypdfium2, cffi, pycparser, and charset-normalizer. Package metadata and each
project's official license file remain authoritative.

Optional profiles are isolated from core:

| Profile | Direct versions | Purpose |
| --- | --- | --- |
| Surya | `surya-ocr==0.21.2` | Official CLI adapter; package code Apache-2.0 |
| PaddleOCR CPU | `paddleocr==3.7.0`, `paddlepaddle==3.3.1` | Official Python result API in a subprocess; package code Apache-2.0 |
| Tesseract | External CLI major 5 | TSV subprocess adapter; code Apache-2.0 |

`requirements-ocr-surya.txt` and `requirements-ocr-paddle.txt` pin their complete resolved
Python graphs with `==`. Tesseract is intentionally not represented by a Python wrapper.
System backends such as llama.cpp, Docker/NVIDIA components, and the Tesseract executable
must be installed and reviewed separately.

## OCR artifacts and terms

Official sources:

- Surya, [official project and CLI documentation](https://github.com/datalab-to/surya).
- Surya 2 GGUF, [official model repository](https://huggingface.co/datalab-to/surya-ocr-2-gguf).
- PaddleOCR, [official general OCR pipeline
  documentation](https://www.paddleocr.ai/latest/en/version3.x/pipeline_usage/OCR.html).
- PaddleOCR, [official PP-StructureV3 pipeline
  documentation](https://www.paddleocr.ai/latest/en/version3.x/pipeline_usage/PP-StructureV3.html).
- PP-OCRv6 medium detection,
  [official model repository](https://huggingface.co/PaddlePaddle/PP-OCRv6_medium_det).
- PP-OCRv6 medium recognition,
  [official model repository](https://huggingface.co/PaddlePaddle/PP-OCRv6_medium_rec).
- Tesseract, [official command-line usage](https://tesseract-ocr.github.io/tessdoc/Command-Line-Usage.html).
- Tesseract language data,
  [official tessdata documentation](https://tesseract-ocr.github.io/tessdoc/Data-Files.html).

Reviewed model identifiers, revisions, file sizes, and checksums are listed in
[OCR](references/ocr.md). That reference also contains the complete Paddle structure-role
table and explicitly marks all six structure artifact families as not independently
reviewed where revision, complete directory size/checksum, and artifact license remain
unknown. Candidate family names are not approval. The manifest must record and preflight
the exact artifact-level facts actually used.

The Surya project states:

- code: Apache-2.0;
- current model weights: modified AI Pubs Open Rail-M;
- free weight use: research, personal use, and startups under USD 5 million in
  funding/revenue;
- broader commercial use: commercial license required from the model provider.

This is a material use restriction. The adapter surfaces it in `SKILL.md`, `ocr-plan`,
preflight output, and [OCR](references/ocr.md). Operator confirmation is required and is not
legal advice.

PaddleOCR and Tesseract code are Apache-2.0. The manifest still records and approves each
specific model or `.traineddata` artifact rather than assuming its terms from the runtime
code. Paddle model directories require a canonical size and inventory checksum in addition
to identifier, immutable revision, official source, and artifact license; unknown structure
facts fail explicit preflight rather than inheriting certainty from PaddleOCR's code
license.
